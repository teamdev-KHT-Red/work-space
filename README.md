## 開発
本プロジェクトでは、メンバーがMVCの各コンポーネントを担当することで開発を進めました。

| メンバー | 担当パート | 役割・実装内容 |
| :--- | :--- | :--- |
| [@TKPRATT](https://github.com/TakeoYabuki) | **Model** | **データロジック担当**<br>盤面データ管理、ホールド機能等、テトリミノのロジック実装。 |
| [@ryoyamadakob106-cmd](https://github.com/ryoyamadakob106-cmd) | **View** | **UI/描画担当**<br>Canvas APIを用いたレンダリング、スコア表示、デザインの実装。 |
| [@h-uhun](https://github.com/h-uhun) | **Controller** | **制御/統合担当**<br>ユーザー入力のハンドリングの実装。 |

3人で効率的に開発を進めるため、以下の理由からMVCを導入しました。
* 役割ごとにファイルを分割し、コンフリクトを減らすため。
* バグ発生時に修正箇所を即座に判断するため。
* コードの可読性、拡張性の向上。
<br>

## チーム開発を通して

ルールとして「機能ごとにブランチを分ける」とはしていましたが、3人とも初めてのチーム開発ということもあり、
コンフリクトや誤ったマージをしてしまい、コードが動かなくなったりというトラブルにも直面しました。<br><br>
その後は、共通の定数ファイルを用意したり、なるべくコンフリクトが発生しないように開発を行いました。<br>
最終的にはGit操作への理解も深まり、エラー発生時の対策、チームで解決する力が身についたと思います。
